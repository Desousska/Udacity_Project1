Work Session 1: The Basics of the Web and HTML

HTML stands for HyperText Markup Language. HyperText is the content you see on a webpage, and markup refers to how that content appears.

Tag: a tag comes in the form of <b> for opening, and </b> for closing. Where "b" in this case, is an instruction that the computer understands to mean "make the font of the text between these tags bold." There are many possible tags, e.g., 'p' for paragraph, or 'br' for linebreak. Note that while most tags require closing tags, some do not (like br, img). One explanation provided for this is that some tags, like img or br do not contain any content, and thus do not need a closing tag to denote where the "ending" is. These are void elements/tags.

Element: Within the brackets [ ] is an HTML Element:[<b> content </b> ].

Attribute: Example: [<a href="http://www.reddit.com"> link </a>] - href in this example is an Attribute. It is a property of the element. Other attributes could be class, id, alt, title, src, etc.

Additional info:

In HTML you use: <!-- ...--> to comment. Commenting is important because it can explain your choices in your native language, instead of in HTML/a computer's (browser's) language

So in the exmaple the word "text" would appear if the image did not load. It is also used as a tool for blind people viewing the webpage. It doesnt need a closing tag because img is a void element. Work Session: 2 Creating Structured Documents with HTML

Boxes:

"Webpages are a bunch of boxes." When you receive a mockup of a site you can take a look and begin cutting up the doc into boxes so that you can translate it to HTML.

HTML's Tree-like Structure:

Since webpages are a bunch of boxes, you can nest smaller boxes within larger boxes.
You can create your first box (or the trunk) with the opening html tag, then 2 branches with head and body tags. Then, the head and body tags (separated now) can branch off into title, div, or paragraph (p) branches, for example. This continues until you have a "tree" of branches, similar to that of a phylogenetic tree which documents ancestry.
For ease of viewing, HTML branches are nested beneath their parent branch. This is achieved by tabbing in 1 level when writing code you wish to keep on the branch of the element before it.
In order to start a new branch you must close the opening tag of the origin of your branch, and all of the open tags within it. For instance, the title tag, then the head tag can be closed, then the body tag opened, creating a new branch.

Avoiding Repetition

Since many branches of the HTML tree have similar elements, it is a good idea to give those similar elements matching class attributes in order to group them together. Later, with CSS, the class can be used to style these similar branches all at once, removing a lot of the potential for repetition in CSS.
To give specific elements unique identifiers you can use ID attributes, and later call them specifically in CSS in case you wish to style that particular branch individually (or even link directly to it).
Avoiding repetition is important because it makes styling with CSS much simpler. By minimizing it, you can make site-wide changes with 1 augmentation to the CSS instead of having to make individual changes to each line of code. It also saves computing power by shortening code, and also makes it much easier for a person to read and debug when looking through the code at a later date.

Work Session 3: CSS

CSS:

  • Cascade: The most specific rule applies
  • Style: Changes affect the look of the page
  • Sheets: You can have any number of CSS sheets

How to approach coding:

  1. Code
  2. Test
  3. Refine

To start coding in HTML/CSS you should:

  1. Boxify: look for natural boxes. You can literally cut up the mock-up page with scissors, or draw boxes around everything
  2. Work on Tags: look for repeated styles and semantic elements. Like in this page, there are "stages" which would likely have the same styling and format, so they are likely going to have repetitive coding behind them. So be sure to group these "like-elements" together to avoid repetitive code
  3. Write the HTML
  4. Apply Styles - from biggest (site-wide) to smallest. Remember that some styles are inherited from parent elements. You can play with developer tools in chrome to determine how styling a particular element will affect the other elements on the page
  5. Box sizing. You need to resize boxes in order for them to all fit nicely on the page together. Keep in mind that there are multiple types of devices people use, so compatibility is an important consideration
  6. Where to position boxes. Positioning matters, and how you determine the position matters. Be sure that if you use css like "flex" that you include the code required to make it compatible with different browsers.
  7. FIX IT!
    • resize windows
    • try different browser types
    • try it with different devices in varying orientations

To Write HTML:

  • use Codepen.io for a quick scratch pad that you can share once you have a free account
  • use Scratchpad.io it's free, no user account needed. Write code and visualize it immediately
  • use Sublime Text 2 to write and save code. The drawback is that you have to open the html in your browser to see it. The plus is that it's all saved on your computer so it is more intuitive
  • use your knowledge of tags to create the boxes you defined while boxifying
  • add the content you want to see

Applying Style by Writing CSS:

  • in CSS, select an element in the following way:
    p{ margin: 10px;}
  • select a class using:
    .description{ color: blue;}
  • select an ID using:
    #price{ color: green;}

Note that in the previous examples, {color: green;}, {color:blue;} or {margin:10px;} are rules. While p, .description, #price are selectors. Color or margin are properties and green, 10px, and blue are values

Box Model & Sizing:

The computed area of an element (i.e., its size) is measured as the sum of the border, padding, and content of the element.

Because this makes knowing the exact size an element will take up on a page difficult (you would have to add all of the above values everytime you change an element's size) use the following format in CSS:
*{
box-sizing:border-box;
}

An important consideration with this is that it is a relatively new way to deal with "boxes" thus it can lead to difficulties when someone is using an old browser. To combat this, you have to add browser specific prefixes. You add these before the attribute, like this:
*{
-webkit-box-sizing:border-box;
-moz-box-sizing:border-box;
-ms-box-sizing:border-box;
box-sizing:border-box
}

You can also set sizes of boxes in terms of percents instead of pixels, to assist in making sizing on the page easier

Box Positioning:

Divs are by default block elements. To make a div fall inline (beside another "box") you can use the CSS rule:
{display: flex;}
Use: display: -webkit-flex; in order to have this work in Safari browsers.
For a full guide to the flex value follow this link.

Fixing code:

How to comment: In CSS,to comment you use:/*...*/
It is important to add comments because it allows you to easily see "why" you chose to do something in your code, which makes debugging it easier. You take the 'code' and elaborate on it using words and descriptions such that it is understandable to an english speaker - and not just to a computer.

Debugging: You can use programs to point out flaws in your code, such as open tags or invalid structuring
HTML Debugger: http://validator.w3.org/. This site allows you to submit your HTML and issues in formatting (e.g., unclosed tags) will be flagged so you can correct them.
CSS Debugger: http://jigsaw.w3.org/css-validator/#validate_by_input. This allows you to do the same for CSS.
